１０２．アクションを統合する

連続したアクション

　前項では、唯一のアクションの実装を行いました。
　しかし、AI操作などで、連続した動きを登録できると便利です。Actionコンポーネントはこうした連続したアクションを登録できるコンポーネントです。
　サンプル102を実行すると以下のように３つの球が連続した動作を繰り返します。

 

図1002a

 

　このサンプルは、回転と移動の連続動作を登録しています。
　Actionコンポーネントは、拡大縮小、回転、移動のラインを別々に持ってます。そのため、回転しながら移動とか縮小しながら回転などの定義が可能です。

　ではソースを見てみましょう。Charactor.hとCharactor.cppに記述があります。まず、ヘッダです。 
    //--------------------------------------------------------------------------------------
    //  class ActionObject : public GameObject;
    //  用途: アクションする配置オブジェクト
    //--------------------------------------------------------------------------------------
    class ActionObject : public GameObject{
    public:
        //構築と破棄
        ActionObject(const shared_ptr<Stage>& StagePtr);
        virtual ~ActionObject();
        //初期化
        virtual void OnCreate() override{}
        void OnCreateWithParam(const Vector3& StartPos);
    };
　宣言はいたってシンプルです。Actionコンポーネントのみ実装する場合、OnUpdate()関数もOnLastUpdate()関数も必要ではありません（ただこれはサンプルだから可能であり、一般的には相互作用を行うので、Update系関数は必要になるでしょう）。
　ただ、Createは、チュートリアル005で紹介したように、OnCreateWithParam()関数を使用します。こうしておくと、オブジェクトの初期位置をメンバに持っておく必要がないので便利です。その場合、OnCreate()仮想関数は、使わなくてもいいので、空関数にしておきます。OnCreateWithParam()関数を使う場合、OnCreateWithParam()関数がOnCreate()仮想関数より先に呼ばれます。

連続したアクションを定義する

　Actionコンポーネントを定義は、Charactor.cppでActionObject::OnCreateWithParam()関数に記述します。以下が、その主要な部分です。 
    //初期化
    void ActionObject::OnCreateWithParam(const Vector3& StartPos){
        //中略
        //アクションの登録
        auto PtrAction = AddComponent<Action>();
        PtrAction->AddRotateBy(1.0f, Vector3(0, XM_PI, 0));
        PtrAction->AddMoveBy(1.0f, Vector3(2.0f, 0, 0));
        PtrAction->AddMoveBy(0.5f, Vector3(0, 1.0f, -1.0f), 
            Lerp::Linear, Lerp::EaseOut, Lerp::Linear);
        PtrAction->AddMoveBy(0.5f, Vector3(0, -1.0f, -1.0f), 
            Lerp::Linear, Lerp::Easein, Lerp::Linear);
        PtrAction->AddMoveBy(2.0f, Vector3(-4.0f, 0, 0));
        PtrAction->AddMoveBy(1.0f, Vector3(0, 0, 2.0f));
        PtrAction->AddMoveBy(1.0f, Vector3(2.0f, 0, 0));
        //ループする
        PtrAction->SetLooped(true);
        //アクション開始
        PtrAction->Run();

        //中略

    }
　ここでポイントは各ライン（拡大縮小、回転、移動）は別々に働くということです。たとえば回転は 
        PtrAction->AddRotateBy(1.0f, Vector3(0, XM_PI, 0));
　と1行記載されてるだけですが、これで、1秒で半回転をループします。移動は全体で6秒の処理ですが、全体でループします。
　もし、回転を1秒で半回転だけ行って、あとは、移動処理時間に合わせるために5秒間何もしない処理を実装する場合、 
        PtrAction->AddRotateInterval(1.0f);
　のように記述します。1秒間、回転しないという意味です。インターバル関数は、拡大縮小や移動にも、それぞれあります。

　また、移動の一部に使用している 
        PtrAction->AddMoveBy(0.5f, Vector3(0, 1.0f, -1.0f), 
            Lerp::Linear, Lerp::EaseOut, Lerp::Linear);
        PtrAction->AddMoveBy(0.5f, Vector3(0, -1.0f, -1.0f), 
            Lerp::Linear, Lerp::Easein, Lerp::Linear);
　は、補間処理を指定しています。Lerp::EaseOutはだんだん遅くなるでLerp::Easeinはだんだん速くなるです。このようにして疑似ジャンプを表現しています。補間処理はこのほかに、3次関数とコサインがあります。ただし、補間を指定できるのは、移動処理だけで、回転や拡大縮小は線形補間のみとなります。

　ActionObjectクラスの記述が終わったら、いつものように、GameStageクラスでインスタンスを実装します。記述方法は、前項までを参考にしてください。
　また、以降のドキュメントでは、GameStageでの各インスタンス作成処理は、特別のことがない限り解説しませんので注意してください。
